# 时间/空间描述要点指引

步骤详情中的 **⏱ 时间要点** 和 **📱 空间要点** 必须覆盖以下维度:

## ⏱ 时间要点

| 类型 | 说明 | 示例 |
|------|------|------|
| 持续时间 | 步骤本身的耗时 | 动画播放 3秒、loading ≤ 2秒 |
| 超时/倒计时 | 用户需在限定时间内完成操作 | 倒计时 30秒超时自动选择 |
| 轮询/刷新 | 系统定期更新的间隔 | 位置刷新 ≤ 5秒 |
| 等待状态 | 用户感知到的加载/等待 | skeleton屏 0.5秒 → 内容 |
| 过渡时间 | 步骤间切换的耗时 | 页面切换动画 300ms |
| 时间约束 | 业务层面的时间规则 | 攻城总耗时不超过3分钟 |

## 📱 空间要点

| 类型 | 说明 | 示例 |
|------|------|------|
| 布局位置 | 元素在屏幕中的位置 | 居中卡片、底部弹出面板 |
| 尺寸/比例 | 元素的大小 | 表单宽 360px、面板占 40% |
| 层级关系 | z-index / 弹窗层级 | 遮罩层上弹窗、Toast 在最上层 |
| 元素排列 | 同一区域内的元素顺序 | 左侧头像+兵力，右侧战力对比 |
| 空间变化 | 交互引起的布局变化 | 展开面板、滑动抽屉、折叠动画 |
| 多屏切换 | 页面/视图之间的跳转 | 全屏列表 → 详情页 → 返回 |
