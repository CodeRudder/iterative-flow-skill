# L1 主流程模板

文件名固定为: `FL-{abbr}-{NN}-{title}-主流程.md`（如 `FL-SIEGE-01-攻城战-主流程.md`）

**设计要求**: 一个主流程文件列出该功能相关的所有主流程链路，提供全貌视图。

---

```markdown
# FL-{abbr}-{NN} {功能名称} 主流程

> **版本**: v{N} | **最后更新**: YYYY-MM-DD
> **流程类型**: L1 主流程（全局路线图）
> **归属功能**: [功能模块描述]
> **触发条件**: [什么触发此主流程开始]
> **结束条件**: [什么标志此主流程结束]
> **前置状态**: [系统/用户在流程开始前的状态]
> **后置状态**: [系统/用户在流程结束后的状态]
> **来源文档**: [PLAN.md](../PLAN.md) 中的原始文档引用

## 关键要求

- [全局性业务约束，如: 攻城战每日限参与3次、单次攻城总耗时不超过3分钟]
- [...]

## 验收标准

- [主流程完成的判定条件，如: 攻城结果已结算、奖励已发放、编队已返回出发城]
- [...]

## 主流程概览

```
FL-{abbr}-{NN} {功能名称}主流程
├── Phase A: {阶段名称}
│   └── → [FL-{abbr}-{NN}](FL-{abbr}-{NN}-{title}.md) {子流程名称}
├── Phase B: {阶段名称}
│   └── → [FL-{abbr}-{NN}](FL-{abbr}-{NN}-{title}.md) {子流程名称}
├── ...
└── Phase N: {阶段名称}
    └── → [FL-{abbr}-{NN}](FL-{abbr}-{NN}-{title}.md) {子流程名称}
```

## 阶段流转

### Phase A → Phase B: {阶段名称}

> **来源**: [PRD X.X 章节](../../path/to/prd.md#x-x)

**进入条件**: [什么条件下从上一阶段进入此阶段]
**阶段目标**: [此阶段完成什么事情]
**对应子流程**: → [FL-{abbr}-{NN}-{title}.md](FL-{abbr}-{NN}-{title}.md)（独立文件，点击查看详情）
**退出条件**: [什么条件下离开此阶段，进入下一阶段]
**异常跳转**:

| 条件 | 跳转至 | 说明 |
|------|--------|------|
| [异常条件] | Phase X / 终止 | [处理说明] |

---

### Phase B → Phase C: {阶段名称}

[同上结构]

---

## 主流程异常汇总

| 触发阶段 | 异常条件 | 处理方式 | 恢复/终止 |
|---------|---------|---------|:---------:|
| Phase A | [条件] | [处理] | Phase X / 终止 |
```
