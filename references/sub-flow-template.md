# 子流程模板（内联于主流程文件）

每个子流程是主流程文件中的一个 `##` 级别章节，不是独立文件。

> **核心原则**: 只写流程（步骤流转、分支、条件、异常），不复制 PRD 的设计内容。设计细节通过引用链接指向 PRD。

---

## 完整示例

以下用攻城战"行军推进"展示一份完整的内联子流程，其中 **FL-SIEGE-02-01 部队出发** 为详细展开步骤，其余步骤省略详情（实际文档中需全部展开）。

```markdown
## FL-SIEGE-02 行军推进

> **来源**: [PRD 3.2 行军系统](../../prd/siege-map.md#3-2) — 仅描述流程，设计细节见 PRD 原文

**触发条件**: 玩家创建攻占任务，编队从出发城市出发
**结束条件**: 编队到达目标城池，弹出攻城确认
**前置状态**: 玩家已选择目标城池，编队已创建，攻占任务已生成
**后置状态**: 编队抵达目标城池附近，等待玩家确认发起攻城

### 流程结构

```
FL-SIEGE-02 行军推进
├── FL-SIEGE-02-01  部队出发
│   ⏱ 出发动画播放 3秒
│   📱 全屏地图视图，部队图标从己方城池弹出
│
├── FL-SIEGE-02-02  实时位置更新
│   ⏱ 位置刷新间隔 ≤ 5秒
│   📱 部队图标沿行军路线平滑移动
│
├── FL-SIEGE-02-03  遭遇拦截
│   ⏱ 到达拦截点时立即触发；屏幕震动 0.5秒
│   📱 红色警告横幅从顶部滑入，地图蒙层半透明遮罩
│
└── FL-SIEGE-02-04  到达目标城池
    ⏱ 到达后自动弹出攻城确认，10秒无操作自动进入
    📱 城池图标放大脉冲动画，底部弹出"发起攻城"按钮
```

### 关键要求

- 行军耗时最短 10秒，最长不超过 1分钟（距离映射为时间）
- 行军过程中玩家可以点击行军精灵查看线路和预计到达时间
- 行军途中可能遭遇拦截，触发拦截处理（FL-SIEGE-02-03）
- 玩家中途取消行军，编队原路返回出发城市（同样受时间限制）

### 验收标准

- 编队精灵从出发城市移动到目标城池
- 到达后自动弹出攻城确认（10秒无操作自动进入）
- 行军途中若被拦截，拦截处理正确触发
- 中途取消时编队按来路返回

### FL-SIEGE-02-01 部队出发

> **来源**: [PRD 3.2.1 部队出发](../../prd/siege-map.md#3-2-1) — 仅描述流程，设计细节见 PRD 原文

**用户感知**: 看到攻占任务列表中该任务状态变为"行军中" → 地图上己方城池弹出部队精灵 → 精灵沿路线开始移动
**关键步骤**: 创建行军实例 → 计算行军路线（避开不可通行区域）→ 计算预计到达时间 → 开始位置轮询更新
**关键要求**: 出发前校验编队状态（未受伤/未在其他任务中），参考 [PRD 3.2.1](../../prd/siege-map.md#3-2-1)
**验收标准**: 行军实例创建成功，部队精灵出现在地图上并开始移动

**⏱ 时间要点**:
- 出发动画播放 3秒（不可跳过）
- 行军总时长 = 距离 × 时间系数，最短 10秒，最长 60秒
- 任务列表状态在出发动画结束后立即更新为"行军中"

**📱 空间要点**:
- 全屏地图视图，中心自动聚焦到出发城池
- 部队精灵（32×32 军旗图标）从城池图标中心弹出，弹出动画 500ms
- 行军路线用虚线绘制，已走过部分（实线），待走部分（虚线）

**异常分支**:
| 条件 | 处理方式 | 转至步骤 |
|------|---------|:--------:|
| 编队已有成员受伤 | 弹出提示"编队中有伤员，是否继续？" → 确认/取消 | FL-SIEGE-02-01（停留） |
| 网络断开 | 本地缓存行军指令，恢复网络后同步 | FL-SIEGE-02-02 |
```

## 步骤详情填写要点

每个步骤（`###` 级别）必须包含以下字段，缺一不可:

| 字段 | 说明 | 常见错误 |
|------|------|---------|
| 来源 | PRD引用链接 | 只写"PRD"不写具体章节链接 |
| 用户感知 | 看到→操作→反馈的三段式 | 只写"显示页面"没有操作和反馈 |
| 关键步骤 | 逻辑/状态变更/调用 | 复制PRD的字段定义和UI布局（应引用） |
| 关键要求 | 本步骤的业务约束 | 写得太笼统如"要正确" |
| 验收标准 | 完成判定条件 | 和"结束条件"混淆 |
| ⏱ 时间要点 | 持续时间/超时/动画 | 写"适当等待"不量化 |
| 📱 空间要点 | 位置/尺寸/层级 | 写"合理布局"不量化 |
| 异常分支 | 条件→处理→跳转 | 只有happy path无异常 |
